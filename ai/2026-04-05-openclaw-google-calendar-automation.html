OpenClaw x Google Calendar

AIエージェントで予定管理を自動化する方法と 403 access_denied の解決策

// VPS からローカル移行、API 取得、OAuth 403 解消までをまとめた Google カレンダー連携ガイド

OpenClaw などの AI エージェントを使って、日常のスケジュール管理を自動化したいと考えたことはありませんか。「明日の予定を教えて」「来週の火曜日に会議を入れて」とテキストで指示するだけで、AI が Google カレンダーを操作してくれたらかなり便利です。

この記事では、OpenClaw と Google Calendar API を連携させるための一連の流れを、VPS からローカル PC への方針転換、API 準備、そしてつまずきやすい 403: access_denied の解決まで含めて整理します。

1. なぜ VPS ではなくローカル PC が向いているのか

24 時間稼働を考えると VPS は魅力的ですが、Google API と AI エージェントの組み合わせでは、まずローカル PC で構築するほうが圧倒的にスムーズです。

  1. OAuth 認証が localhost 前提で進む
    Google API の初回認証では、ブラウザでログイン後に localhost へ戻されるフローが一般的です。VPS 上で実行すると、手元のブラウザから VPS 側の localhost に到達できず、認証完了までが面倒になります。
  2. データセンター IP は bot 判定を受けやすい
    Playwright などを使ってブラウザ操作を伴う場合、VPS の IP は Google 側に不審判定されやすく、ログインや認証フローが不安定になりがちです。自宅回線や通常の Wi-Fi 環境のほうが安全に進められます。

2. Google Calendar API の準備

ステップ1：API を有効化する

Google Cloud Console で新しいプロジェクトを作成し、「API とサービス」→「ライブラリ」から Google Calendar API を有効化します。

ステップ2：OAuth クライアント ID を作成する

「認証情報」タブから「認証情報を作成」→「OAuth クライアント ID」を選択し、アプリケーション種別は デスクトップ アプリ を指定します。作成後にダウンロードできる credentials.json が、OpenClaw 側で使う認証情報になります。

3. 最大の罠：403 access_denied の正体

ここで多くの人が詰まるのが OAuth 同意画面の設定不足です。認証時に次のようなエラーが表示されることがあります。

アクセスをブロック: [アプリ名] は Google の審査プロセスを完了していません

[アプリ名] は現在テスト中で、デベロッパーに承認されたテスターのみがアクセスできます。
エラー 403: access_denied

原因

自作アプリは、Google の審査を通していない限りデフォルトで 「テスト中」 扱いになります。この状態では、OAuth 同意画面で事前登録した テストユーザー しかログインできません。

解決手順

  1. Google Cloud Console で 「OAuth 同意画面」 を開く。
  2. 下部の 「テストユーザー」 セクションを確認する。
  3. 「ADD USERS」 から、自分の Google アカウントを追加して保存する。

設定後は数分待ち、ローカルに残っている古い token.json を削除してから、もう一度認証をやり直すのが安全です。これで 403 が解消されるケースがほとんどです。

4. OpenClaw に実行させる流れ

403 エラーが解消されたら、ダウンロードした credentials.json を OpenClaw のワークスペースに配置します。あとは自然言語で次のように依頼できます。

ワークスペースにある credentials.json を使って、Google Calendar API 経由で予定を追加・取得する Python スクリプトを作成し、実行して。

初回のみローカル PC のブラウザが立ち上がり、Google の認証を求められます。許可すると token.json が生成され、以降は「予定を追加して」「明日の予定を見せて」といった自然言語の指示でカレンダー操作を自動化しやすくなります。

5. まとめ

Google API 連携で重要なのは、複雑な実装テクニックよりも 実行環境と OAuth 設定を間違えないこと です。特に、「ローカル PC で動かす」「OAuth 同意画面でテストユーザーを追加する」 の 2 点を押さえるだけで、セットアップ難易度はかなり下がります。

カレンダー連携が安定すると、Web から集めた情報を自動で予定化したり、エージェント経由で日程調整を進めたりと、より実用的な自動化へ広げやすくなります。OpenClaw を実務レベルで活かしたいなら、まずはこの構成から始めるのがおすすめです。
