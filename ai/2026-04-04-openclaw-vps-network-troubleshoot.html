OpenClaw Network Troubleshoot

VPS 環境におけるパケットフィルターの罠と解決策

// XServer VPS 上で browser-js が fetch failed になる問題をインフラ起点で切り分け

関連用語

OpenClaw や VPS の意味を先に確認したい場合は、用語集から見られます。

1. 発生した現象と環境

  • 対象: OpenClaw v2026.4.1（XServer VPS / Ubuntu）
  • 現象: skills check は Ready だが、外部サイトアクセスがタイムアウト / fetch failed。
  • 補足: curl は無反応、nslookup のみ成功。

2. 主因：VPS 固有のパケットフィルター

根本原因は OS 内部ではなく、VPS 事業者側のパケットフィルター仕様でした。フィルター ON 時は許可ポート以外が双方向遮断されます。

  • DNS（UDP 53）が通るケースはある。
  • HTTP/HTTPS（TCP 80/443）は戻りパケットが遮断され失敗しうる。

3. 解決ステップ

ステップ1：Scope 権限を解放

openclaw devices list
openclaw devices approve <REQUEST_ID>

ステップ2：DNS の安定化

echo "nameserver 8.8.8.8" > /etc/resolv.conf

ステップ3：インフラ・フィルターを開放（最重要）

  1. パケットフィルターを一時 OFF。
  2. または TCP 80/443 を許可ポートに追加。

4. 診断コマンド

nslookup google.com
curl -v --connect-timeout 5 https://www.google.com

5. 結論

今回のボトルネックは LLM ではなく、ネットワーク・インフラ制約でした。軽量モデルでも、インフラ要件が整えば十分にツール運用可能です。
