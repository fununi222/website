altra-recommender
制作記録

"ロジックと UI の両立を学んだ Web アプリ制作のふりかえり"

過去に制作した altra-recommender は、ユーザーごとに合いそうな候補を提示することを意識して作ったレコメンド系の Web アプリです。見た目だけではなく、「どうやって候補を出すか」 と 「どう見せれば使いやすいか」 の両方を考える必要があり、ロジックと UI をつなぐ難しさを学べた制作でした。

1. 制作背景

一覧をそのまま見せるのではなく、利用者に合わせて情報を絞り込んだり、候補を提案したりする仕組みに興味があり、このアプリに取り組みました。普段使っているサービスでは当たり前に見えるレコメンド機能も、自分で作る側に回ると、データの扱い方、条件分岐、結果の見せ方まで含めて考えることが多くあります。

2. 制作で意識したこと

Experience-First

レコメンド機能は裏側で正しい結果を返せば終わりではありません。画面上でどう受け取られるかを考えながら実装しました。

Balance

情報が多すぎると選びづらくなります。どの情報を先に見せるか、一覧性と理解しやすさのバランスを意識しました。

3. 制作を通して学んだこと

フロントエンドだけでなく、ロジック設計まで含めてアプリ全体を見る視点が身についたことです。特に、推奨結果の理由や納得感（Explainability）まで含めて設計する重要性を、この制作で実感しました。

4. 今ならこう改善する

  • 推薦の理由を UI 上で伝え、納得感を高める
  • データ構造とロジックを整理し、保守しやすくする
  • 結果一覧の見せ方を改善し、比較しやすい画面にする
