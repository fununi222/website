# AIを最強のシニアエンジニアに変える｜即戦力コードレビュー・プロンプト集

「AIにレビューを頼んでも『概ね良好です』としか言われない」
「肝心なバグを見逃されて、結局自分で全部見直している……」

それはAIの能力不足ではなく、**「命令（プロンプト）の解像度」**が低いからです。AIは指示が具体的であればあるほど、鋭いメスを入れるシニアエンジニアへと変貌します。

本記事では、Java, JavaScript, Pythonなどの主要言語において、**「静的解析ツールが沈黙する、実装の意図に潜むバグ」**を100%近く特定させるための、門外不出のプロンプトテンプレートを公開します。

---

## 1. 【基礎】AIの役割（ペルソナ）を固定せよ

プロンプトの冒頭に、必ず以下の「人格」を与えてください。これだけで回答の質が2段階上がります。

> **「あなたは、15年の経験を持つシニアソフトウェアエンジニアであり、特にセキュリティとパフォーマンス最適化のスペシャリストです。これから提示するコードに対し、単なる構文チェックではなく、アーキテクチャの妥当性、エッジケースの考慮、保守性の観点から、厳格かつ丁寧なレビューを行ってください。」**

---

## 2. 言語別・目的別の「神プロンプト」テンプレート

### A. Java: リソース管理と例外処理の徹底
Java開発者が最もハマる `try-with-resources` 漏れや、例外のシャドウイングを暴きます。

```markdown
以下のJavaコードにおいて、AutoCloseableなリソース（DB接続、ストリーム等）が確実に解放されているか確認してください。
特に、finally節での手動closeによる例外の隠蔽（元のエラーが上書きされる現象）がないかチェックし、
try-with-resources形式へのリファクタリング案を、パフォーマンスへの影響も含めて提示してください。
```

### B. JavaScript/React: メモリリークとHooksの罠
[SPA](https://fununi222.github.io/website/html/glossary/system-glossary.html#:~:text="SPA")で致命傷となる `useEffect` のクリーンアップ漏れを特定します。

```markdown
このReactコンポーネントのuseEffect内で登録されたイベントリスナーやタイマーが、
アンマウント時に適切にクリーンアップされているか検証してください。
ライフサイクルを超えて生存し続け、メモリリークを引き起こす可能性のあるクロージャを特定し、
依存配列（dependency array）の妥当性についても言及してください。
```

### C. 全言語共通: セキュリティ監査
SQLインジェクションや、機密情報のログ出力などの「うっかりミス」を防ぎます。

```markdown
このコードに含まれるセキュリティ上の脆弱性を、OWASP Top 10の基準で監査してください。
特に、ユーザー入力のサニタイズ漏れ、ハードコードされた認証情報、
意図しない例外情報の外部出力がないかを精査し、修正案を提示してください。
```

---

## 3. レビュー精度を極限まで高める「もう一言」

回答の最後に、以下のフレーズを付け加えることで、AIはより深く思考します。

*   **「このコードが数百万ユーザーのトラフィックに耐えられるか、懸念点を挙げてください」**
*   **「私のようなジュニアエンジニアが読み間違える可能性のある、難解な箇所を指摘してください」**
*   **「この変更によって、既存のユニットテストが壊れる可能性のある箇所はありますか？」**

## 4. まとめ：プロンプトはエンジニアの「知性」の投影

AIレビューは、魔法ではありません。あなたが「どこを気にするべきか」を正しくAIに伝えられた時、AIはあなたの知性を補完する最強のパートナーになります。
