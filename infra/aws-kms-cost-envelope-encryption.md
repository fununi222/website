# 【記事③】AWS KMSのAPI課金でクラウド破産!? 暗号化コストを下げる「エンベロープ暗号化」とVault設計

## 導入
「AWSのデータを安全のためにKMSで暗号化したら、毎月数十万円のAPI利用料が請求された…」

クラウドインフラを本格運用し始めた企業から、悲鳴のように聞こえてくるのがこの**「KMSのAPI課金トラップ」**です。

前回の記事（[記事②：AWSバックアップをオンプレに引く罠](https://fununi222.github.io/website/article.html?md=infra/aws-egress-cost-aurora-benefits.md)）で、データをAWS内の別アカウントに隔離（Vault化）する手法を解説しました。隔離するデータを暗号化するのは鉄則ですが、**AWS標準の暗号化の仕組みを何も考えずに使うと、とんでもない請求が来ます。**

この記事では、KMS課金が爆発する理由と、それを数ドルのコストに抑え込む魔法の設計「エンベロープ暗号化」、そして混同されがちな「3つのVault（金庫）」の違いを解説します。

## 1. なぜKMSで「クラウド破産」が起きるのか？

AWS KMS（Key Management Service）の料金体系は以下の2つです。

1.  **鍵の保管料:** 月額約150円（安い）
2.  **API呼び出し料:** 1万回あたり約4.5円（**ここが罠！**）

「1万回で4.5円なら安いじゃないか」と思いますよね。
しかし、システムが**「データベースにデータを1行書き込むたびに、KMSを直接呼び出して暗号化する」**という設計（直接暗号化）になっていたらどうなるでしょうか？

月間10億回のアクセスがあるシステムの場合、通信回数に比例して課金され、**毎月数十万〜数百万円**が青天井で請求されることになります。

## 2. 勘違い注意：無料の「Vault」に逃げるのはNG

このKMS課金を逃れるために、「無料（OSS）のHashiCorp Vaultを自作サーバーで動かして、定額で暗号化させよう」と考えるエンジニアがいます。

**結論、これは絶対にやめてください。**

自作したVaultサーバーがダウンすれば、システム全体が復号できなくなり完全停止します。月に数十万円をケチるために、優秀なエンジニアが24時間サーバーの監視と保守（お守り）をするのは、コストパフォーマンスが最悪です。

## 3. 解決策：劇的に安くなる「エンベロープ暗号化」

自前のサーバーを立てずとも、AWS標準の仕組みだけでKMSのコストをほぼゼロにするベストプラクティスがあります。それが**「エンベロープ（封筒）暗号化」**です.

### 仕組み（なぜ安くなるのか？）
1.  毎回KMSにお願いするのではなく、KMSから**「使い捨てのデータキー」**を1つだけ発行してもらいます。
2.  システムは、その使い捨てキーを数分〜数時間、自分のメモリ内に保存（キャッシュ）します。
3.  実際のデータの暗号化は、KMSと通信せず、**自分のサーバーの中だけで（無料で）**行います。

この設計なら、KMSと通信するのは「使い捨てキーを発行する数十分に1回」だけになるため、月額料金は数百円レベルまで激減します。

[公式情報：AWS KMS エンベロープ暗号化の概念](https://docs.aws.amazon.com/ja_jp/kms/latest/developerguide/concepts.html#enveloping)

## 4. 補足：IT業界で混同される「3つのVault」の違い

暗号化に関連して、データを隔離する「Vault（金庫）」という言葉もよく混乱を招きます。以下の3つのは全くの別物です。

* **HashiCorp Vault:** パスワードなどを管理する「小さな貴重品ロッカー」。大量のバックアップデータは入りません。
* **Rubrik Cloud Vault:** Rubrik社が用意する、オンプレミスデータを退避させるための「巨大なクラウド貸し倉庫」。
* **AWS Backup Vault（クロスアカウント）:** AWSの中で、RDSなどのバックアップを隔離するための「論理的な金庫（別アカウント）」。

AWS上のRDSを守る場合、Rubrik等を用いて**「別アカウントをVault（金庫）として使い、中身をエンベロープ暗号化で安価に守る」**のが最強のセキュリティアーキテクチャとなります。

## まとめ：正しいインフラ設計にはプロの知見が必須
* 毎回KMSを呼び出す直接暗号化は、高額請求の元。
* HashiCorp Vaultの自作運用は、障害と人件費の元。
* 「エンベロープ暗号化」と「クロスアカウント隔離」の組み合わせが正解。
