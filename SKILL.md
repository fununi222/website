# FunUni-lab: Blog Maintenance & Technical Insights

このドキュメントは、FunUni-labブログの技術的な選定理由やメンテナンス手法をまとめた非公開ドキュメント（SKILL.md）です。次回以降の更新時に技術的な一貫性を保つためのリファレンスとして活用してください。

## 1. デザインシステムの設計思想 (Premium Design)

### Glassmorphism (ガラス質感)
- **選定理由**: モダンで先進的な印象を与えるため、`backdrop-filter: blur(20px)` を多用したデザインを採用。
- **実装のコツ**: 背景色を `rgba(15, 23, 42, 0.6)` などの半透明に設定し、1pxの淡いボーダー（`rgba(255, 255, 255, 0.08)`）を加えることで、境界が美しく浮かび上がります。

### Typography (タイポグラフィ)
- **フォント**: `Inter`（英数・モダン・可読性）と `Noto Sans JP`（日本語・安定性）を組み合わせ、さらにコードブロック用に `JetBrains Mono` を採用。
- **階層化**: 見出しには `clamp()` 関数を用いることで、デスクトップからモバイルまでシームレスにフォントサイズが最適化されるように設定。

### Interactive Reports (対話的レポート)
- **特徴**: 静的な記事だけでなく、`Chart.js` による視覚化や、Tailwind CSS を直接活用した動的な UI（タブ、フェードイン、タイピング文字など）を採用。
- **選定理由**: 複雑な概念（AIの推論プロセスやモデルの使い分け）を視覚的に分かりやすく伝え、体験価値を高めるため。
- **実装のコツ**: デザインの柔軟性を優先し、全画面でのレイアウトが必要な場合は `style.css` に依存せず、直接 Tailwind のユーティリティクラスを使用して、各記事に最適化されたパレット（例：Warm Sand & Teal）を適用します。

## 2. 技術的な更新内容と根拠

### VMware CBT/CTK不整合の是正
- **事象の詳細**: CBTが不整合を起こすと、増分バックアップであるはずがフルスキャン（事実上のフルバックアップ）に転向し、ストレージ容量の枯渇やバックアップ時間の増大を招きます。
- **是正のメカニズム**:
    1. `ChangeTrackingEnabled = $true` を再設定。
    2. `New-Snapshot` ＆ `Remove-Snapshot` を実行。これによりVMのリコンフィグがコミットされ、CTKファイルが正しく生成・初期化されます。
- **スクリプトのポイント**: `PoweredOn` 状態のVMのみを対象とすることで、停止中のVMに対する不要な操作（スナップショット作成エラー等）を回避します。

### Jenkins連携システム
- **構成**: Jenkinsエージェント（PowerCLIインストール済み）で1日1回の周期実行。
- **目的**: バックアップ実行前に事前に構成を正常化しておくことで、「バックアップ失敗後の対応」から「失敗させないための能動的な運用」へとシフトするためです。

## 3. 次回更新時のガイドライン

1. **記事の追加**: 新しい記事は `it/` などのカテゴリフォルダに格納し、`index.html` およびカテゴリトップの `index.html` にリンクを追加してください。
2. **デザインの維持**: 見出し、本文、コードブロックには共通の `style.css` をリンクし、一貫性を保ってください。
3. **非公開情報の管理**: リポジトリのルートに `.gitignore` が配置されており、この `SKILL.md` はGitにアップロードされないように設定されています。機密性の高い技術メモなどはこのファイルに追記してください。
