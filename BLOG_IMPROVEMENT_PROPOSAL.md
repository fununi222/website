# ブログ改善提案（初回診断）

> 本ドキュメントは、ブログ改善提案の単一ソースです。SKILL.md に重複していた改善計画の記載は本書へ統合しました。

更新日: 2026-04-04

## 1. 現状構成サマリ

- ルートに `index.html` と `style.css` があり、全カテゴリで共通デザインを利用。
- カテゴリは `it/`, `ai/`, `points/`, `lpo/` に分離され、それぞれに `index.html` を配置。
- 記事は主に HTML で公開され、`it/` と `points/` は一部 Markdown 原稿（`.md`）も併存。
- `lpo/` は通常ブログ記事ではなく、ダッシュボード形式の独立コンテンツ。

## 2. 強み

1. **カテゴリ導線が明快**: トップページから各領域に直接遷移できる。
2. **統一UIが完成している**: `style.css` でトーン＆マナーが統一されている。
3. **テーマの一貫性**: インフラ運用・AI活用・ポイ活最適化が「生活を技術で改善する」軸で繋がっている。

## 3. 主要課題（優先度順）

## P1: 情報設計（回遊性）の不足

- 記事ページに「関連記事」「前後記事」「カテゴリ内一覧」がなく、読み切りで離脱しやすい。
- トップページに新着一覧が少なく、過去記事の発見性が低い。

## P1: SEO 基礎実装が不足

- 多くのページで `meta description` が未整備。
- OGP/Twitter Card 未実装。
- 構造化データ（JSON-LD: BlogPosting, BreadcrumbList）が未導入。

## P2: 制作運用の二重管理コスト

- `.md` と `.html` の二重編集が必要な状態があり、差分ズレのリスクがある。
- 例: 同一テーマでも md/html の内容粒度に差が出ているページが存在。

## P2: 計測基盤不足

- 記事ごとの流入・離脱・CV（回遊/問い合わせ等）を追いにくい。

## 4. まず着手すべき改善（最短2週間）

### Step 1（即日〜3日）: 低コストSEO改善

- 全記事に `meta description` を追加。
- OGP/Twitter Card をテンプレート化して実装。
- `sitemap.xml` / `robots.txt` を追加。

### Step 2（3日〜7日）: 回遊導線の追加

- 記事下に「関連記事3件」「カテゴリ一覧へ戻る」を追加。
- カテゴリ index に投稿日順カード（最低6件まで）を表示。
- トップに「最新記事一覧（全カテゴリ横断）」を追加。

### Step 3（1〜2週）: 運用の単一ソース化

- 原稿は Markdown を正とし、HTMLを自動生成する仕組みへ移行。
- frontmatter（title/date/category/description/tags）を導入。

## 5. 30日ロードマップ（提案）

- **Week 1**: メタ情報・OGP・サイトマップ整備。
- **Week 2**: 回遊導線（関連記事・新着）実装。
- **Week 3**: md→html 生成フロー（スクリプト化）導入。
- **Week 4**: 計測導入（PV、スクロール、回遊率）と改善サイクル化。

## 6. KPI（改善効果の判定指標）

- 平均滞在時間: +20%
- 1セッションあたり閲覧ページ数: +30%
- 検索流入比率: +15%
- 直帰率: -10%

## 7. 次アクション

次フェーズでは、以下を実施するのが最も効果的です。

1. `meta/OGP` 実装対象ページの棚卸し。
2. 共通テンプレート（head / 記事末導線）の定義。
3. md→html 生成スクリプト方針の決定（簡易スクリプト or 静的サイトジェネレータ）。
